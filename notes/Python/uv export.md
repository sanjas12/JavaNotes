Для экспорта зависимостей из `pyproject.toml` в `requirements.txt` 

Разработчики `uv` **рекомендуют избегать постоянного использования `requirements.txt`** наряду с `uv.lock`. Файл блокировки `uv.lock` гораздо более функционален, и многие его возможности невозможно передать в `requirements.txt`. Команда `export` рассматривается скорее как "мост" для взаимодействия с экосистемой `pip`, а не как основной способ управления зависимостями в проектах, использующих `uv`.

`uv export` — это команда для преобразования файла блокировки `uv.lock` в другие форматы. Она нужна, чтобы интегрировать `uv` с инструментами, которые пока не поддерживают его родной формат, например, с `pip` или системами анализа безопасности.

Вот основные сведения о её использовании.

| Флаг                    | Эффект                                                                                           |
| ----------------------- | ------------------------------------------------------------------------------------------------ |
| `--no-dev`              | Исключает dev-зависимости                                                                        |
| `--no-editable`         | Исключает editable-установку текущего проекта                                                    |
| `--no-emit-project`     | Полностью исключает текущий проект (и `-e .`, и обычную установку)                               |
| `--frozen`              | Использует существующий `uv.lock` без проверок                                                   |
| **`--format`**          | Выбор выходного формата (см. таблицу выше).                                                      |
| **`-o, --output-file`** | Запись результата в файл, а не в `stdout` (терминал).                                            |
| **`--no-dev`**          | Исключает dev-зависимости из вывода. Полезно для продакшн-сборок.                                |
| **`--no-hashes`**       | Исключает хэши (`# hash=...`) из `requirements.txt` для совместимости со старыми версиями `pip`. |

## Основная команда

```bash
 uv export --format requirements.txt --no-emit-project --no-hashes --group build > requirements.txt 
```

**Только основные зависимости** (без build-группы):
```bash
uv export --no-hashes --no-dev -o requirements.txt
```

**Включить build-группу** (`cx-freeze` и др.):

```bash
uv export --no-hashes --group build -o requirements.txt
```

**Без маркеров платформы** (если нужен универсальный файл):

```bash
uv export --no-hashes --no-emit-markers -o requirements.txt
```

---

## Установка на машине без uv

После генерации файла на целевой системе достаточно:

```bash
pip install -r requirements.txt
```

### 📦 Поддерживаемые форматы

`uv export` умеет создавать файлы в трёх основных форматах:

| Формат                 | Описание                                                                                             | Основное применение                                                                    |
| :--------------------- | :--------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------- |
| **`requirements.txt`** | Традиционный формат, совместимый с `pip`. Это стандартный формат при вызове `uv export`.             | Развертывание проектов в средах, где используется только `pip`, без установки `uv`.    |
| **`pylock.toml`**      | Стандартизированный формат файлов блокировки для Python, описанный в **PEP 751**.                    | Использование в будущих инструментах, которые будут поддерживать этот новый стандарт.  |
| **`cyclonedx1.5`**     | Формат **Software Bill of Materials** (SBOM), отраслевой стандарт для инвентаризации компонентов ПО. | Аудит безопасности, анализ лицензий и обеспечение прозрачности цепочки поставок (SCA). |
