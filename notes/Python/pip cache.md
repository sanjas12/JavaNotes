Вот основные команды для работы с кэшем pip:

## Просмотр информации о кэше

```bash
# Показать информацию о кэше (размер, расположение)
pip cache info

# Список всех пакетов в кэше
pip cache list
```

## Управление кэшем

```bash
# Удалить конкретный пакет из кэша
pip cache remove numpy

# Очистить весь кэш pip
pip cache purge

# Удалить устаревшие/неиспользуемые кэшированные версии
pip cache remove --no-cache-dir
```

## Где находится кэш pip

**По умолчанию:**
- **Linux/macOS:** `~/.cache/pip`
- **Windows:** `%LocalAppData%\pip\cache`
- **macOS (старые версии):** `~/Library/Caches/pip`

## Отключение кэша

```bash
# Временно отключить кэш для текущей установки
pip install package --no-cache-dir

# Постоянно отключить (через конфиг)
pip config set global.no-cache-dir true

# Удалить настройку отключения
pip config unset global.no-cache-dir
```

## Полезные команды

```bash
# Проверить размер кэша
pip cache info | grep "Total size"

# Очистить кэш для всех пакетов старше N дней (нужно скриптом)
find ~/.cache/pip -type f -atime +30 -delete  # Linux/macOS

# Показать все настройки кэша
pip config list | grep cache
```

## Быстрая очистка (для вашего проекта)

Если кэш занимает много места и нужна полная очистка:

```bash
pip cache purge
```

Или вручную удалить папку кэша:
- **Windows:** `rmdir /s /q %LocalAppData%\pip\cache`
- **Linux/macOS:** `rm -rf ~/.cache/pip`

## Когда очищать кэш?

1. **Проблемы с установкой** - пакет повреждён в кэше
2. **Освобождение места** - кэш может занимать несколько ГБ
3. **Принудительная загрузка свежей версии** - если на PyPI обновили пакет

**Совет:** Обычно кэш полезен и его не нужно трогать, если нет проблем с установкой или не хватает места на диске.
