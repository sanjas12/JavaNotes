`git diff` показывает различия между коммитами, ветками, индексом (staging area) и рабочим каталогом. Вот основные варианты использования:

## 1. **Неподготовленные изменения**
Показывает изменения в рабочем каталоге, которые ещё не добавлены в индекс:
```bash
git diff
```

## 2. **Подготовленные изменения (staged)**
Показывает изменения, которые уже добавлены в индекс (`git add`), но ещё не закоммичены:
```bash
git diff --staged
# или
git diff --cached
```

## 3. **Между коммитами**
```bash
# Между двумя коммитами (по хэшам или ссылкам)
git diff commit1 commit2

# Между HEAD и предыдущим коммитом
git diff HEAD HEAD~1

# Между двумя ветками
git diff main feature-branch
```

## 4. **Рабочий каталог vs коммит**
```bash
# Отличия текущего состояния от коммита
git diff commit

# Отличия от предыдущего коммита
git diff HEAD~1

# Отличия от коммита 2 шага назад
git diff HEAD~2
```

## 5. **Для конкретных файлов**
```bash
# Изменения только в определённых файлах
git diff -- файл1 файл2

# Staged изменения конкретного файла
git diff --staged -- файл

# Между коммитами для конкретного файла
git diff commit1 commit2 -- файл
```

## 6. **Ограничение вывода**
```bash
# Только имена изменённых файлов
git diff --name-only

# Статистика изменений (+, - строки)
git diff --stat

# Краткий список изменённых файлов
git diff --name-status

# Компактный вид
git diff --compact-summary
```

## 7. **Игнорирование пробелов**
```bash
# Игнорировать изменения в пробелах
git diff --ignore-space-change
git diff --ignore-all-space
```

## 8. **Графическое представление**
```bash
# Цветной вывод (обычно включен по умолчанию)
git diff --color

# Использовать внешний инструмент сравнения
git difftool
```

## 9. **Между ветками с общим предком**
```bash
# Показать изменения в текущей ветке относительно main
git diff main...HEAD

# Или короче (если вы на feature-ветке)
git diff main...
```

## 10. **Сравнение с удалённой веткой**
```bash
# Перед push: что изменится в удалённом репозитории
git diff origin/main

# Сравнить с удалённой веткой
git diff origin/main feature-branch
```

## Практические примеры:

```bash
# Что я изменил, но ещё не добавил в git add?
git diff

# Что уже в индексе, но ещё не закоммичено?
git diff --staged

# Все изменения (и staged, и unstaged) относительно последнего коммита
git diff HEAD

# Показать только имена файлов, которые изменятся при merge
git diff --name-only main feature-branch

# Сравнить текущий файл с версией из старого коммита
git diff abc123def -- путь/к/файлу

# Посмотреть изменения за последние 2 коммита
git diff HEAD~2 HEAD
```

## Полезные комбинации:

```bash
# Показать изменения в staged файлах кратко
git diff --staged --stat

# Сравнить две ветки с цветом и статистикой
git diff --color --stat main develop

# Игнорировать изменения в пробелах и показать только имена файлов
git diff --ignore-all-space --name-only
```

**Совет:** Используйте `git difftool` для открытия изменений в графическом инструменте (например, VSCode, Meld, Beyond Compare), если настроен соответствующий инструмент.
